## Anatomy

The input group is split into three parts. `InputGroup` is the required container (it provides the shared visual shell). `InputGroupAddon` is a fixed-size slot for non-input content. `InputGroupDivider` is an optional vertical separator between logical sections.

```tsx
// Icon + TextInput
<InputGroup>
  <InputGroupAddon><SearchIcon /></InputGroupAddon>
  <TextInput placeholder='Search...' />
</InputGroup>

// Color chit + ColorInput
<InputGroup>
  <InputGroupAddon>
    <ColorChit color={color} />
  </InputGroupAddon>
  <ColorInput value={color} onValueChange={setColor} />
</InputGroup>

// ScrubArea + NumericInput (Root wraps Group)
<NumericInputRoot defaultValue={50} min={0} max={100}>
  <InputGroup>
    <InputGroupAddon>
      <NumericScrubArea className='text-grey-500 flex size-6 items-center justify-center'>
        X
      </NumericScrubArea>
    </InputGroupAddon>
    <NumericInput />
  </InputGroup>
</NumericInputRoot>
```

### Multi-input Groups

Combine multiple input types in one group, separated by dividers:

```tsx
// Color + opacity
<InputGroup>
  <InputGroupAddon><ColorChit color={color} /></InputGroupAddon>
  <ColorInput value={color} onValueChange={setColor} />
  <InputGroupDivider />
  <NumericInputRoot defaultValue={100} min={0} max={100}>
    <NumericInput />
  </NumericInputRoot>
  <InputGroupAddon>%</InputGroupAddon>
</InputGroup>
```

### Integration with NumericInputRoot

Two patterns depending on whether the group contains one or multiple numeric fields:

- **Single numeric**: `NumericInputRoot` wraps `InputGroup` from outside
- **Multi numeric**: Each `NumericInputRoot` sits inside `InputGroup` as a child

`NumericInputRoot` automatically strips its own ring/background when it detects it's inside an `InputGroup`.
